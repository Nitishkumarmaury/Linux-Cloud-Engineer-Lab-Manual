# Linux Practicals for Cloud Engineers

This repository contains hands-on practical exercises for cloud engineering, focusing on essential Linux skills required in cloud environments. Each practical is designed to build your expertise in server management, security, deployment, and cloud operations.

## Prerequisites

- A Google Cloud Platform (GCP) account
- Basic familiarity with command line
- Git installed on your local machine
- SSH client (built into Linux/Mac, PuTTY or Windows Terminal for Windows)

## Repository Structure

```
.
├── practicals/
│   ├── practical-1/   # Server Setup & Hardening
│   ├── practical-2/   # Nginx Web Server Deployment
│   ├── practical-3/   # User & Group Management
│   ├── practical-4/   # Backup Automation
│   ├── practical-5/   # Log Monitoring
│   ├── practical-6/   # Service Troubleshooting
│   ├── practical-7/   # Performance Monitoring
│   ├── practical-8/   # Firewall Configuration
│   ├── practical-9/   # Docker Basics
│   └── practical-10/  # Multi-VM Application
└── scripts/          # Utility scripts used in practicals
```

## Getting Started

1. Clone this repository:
```bash
git clone https://github.com/[your-username]/linux-cloud-practicals.git
cd linux-cloud-practicals
```

2. Each practical has its own directory with:
   - README.md with step-by-step instructions
   - Code files and scripts where applicable
   - Documentation of expected outcomes
   - Troubleshooting guides

3. Follow the practicals in order - they build upon each other

## Learning Path

1. **Server Basics & Security**
   - VM provisioning
   - SSH configuration
   - User management
   - Security hardening

2. **Web Services & Deployment**
   - Web server setup
   - Service management
   - Basic networking

3. **System Administration**
   - User/Group management
   - Permission systems
   - Project organization

4. **Automation & Monitoring**
   - Bash scripting
   - Cron jobs
   - Log analysis
   - Performance monitoring

5. **Advanced Topics**
   - Containerization with Docker
   - Multi-VM architectures
   - Database servers
   - Firewall configuration
